Website Design: vivek.dev

Portfolio by Vivek Selvaraj


Interface

I wanted my website to reflect my principles and taste. It had to be simple, responsive, privacy-first and user-centric.

Colour

To be responsive and user-centric, I should respect the user's preferred theme. The theme of the site changes along with the system theme.

I wanted to be subtle with a bright and warm colour for the light theme. And for the dark theme, I went with a midnightish dark grey.

#FCF9F8
#222830

Accent Colour

I needed a brand colour. I like shades of green and blue. On exploring, I came across cornflower blue. Something about the hue, maybe the familiarity, or the tranquillity of it, I chose cornflower blue to be my accent shade.

#6495ED

I kept the accents within the hyperlinks aligning with my idea of keeping it simple.

Typography

One other primary goal for my website is to be super fast. So fast as if it was pre-installed. To achieve that, I've to cut down on loading resources as much as possible.

A typical font file is around 50kB. And my home page is just around 30kB (27kB is taken up by my favicon).

Thus, I decided to stick with the system fonts. I prefer sans serif to serif. So, I added all my preferred sans serif typefaces in my CSS, falling back to the system sans serif font.

Images

Images are resource-heavy, and I try to refrain from them as much as possible.

But a portfolio without images would be bland. I've added images as it is in the portfolio for the time being.

I am exploring efficient alternatives like dithered images and images with lower colour depth to replace current data-rich assets. This would significantly reduce the image bundle sizes.

Navigation

A decluttered homepage and content pages resulted in more straightforward navigation.

I removed headers from all of the content pages. To go back, I gave a link in the subtitle.

Build System and Hosting

To build my website, I was on the lookout for the perfect website builder, but I couldn't find one to my liking. In vain, I decided to code one myself from the ground up.

Writing one from scratch is okay, but it becomes a maintenance nightmare in the longer run. Some sort of a build pipeline is needed to ease maintainability.

This was when I found Publish - a static site generator in Swift. Publish takes the load of converting the markdown files to structured HTML, and it is also flexible enough for me to write my plugins. With my custom CSS, it looks just the way I envisioned.

Hosting

The website is hosted on GitHub pages with my custom domain.

Next Steps

I found that cornflower blue on my light background fails in contrast ratio. Though my site scored a 100 on the best practices in the Lighthouse report, it scored only a 93 on accessibility.

My next step is to score a 100 on accessibility.

As I stated before, I am writing a Publish plugin to optimise all the image assets just before deployment.
